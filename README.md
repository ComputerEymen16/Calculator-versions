# Calculator-versions
 Calculator 1.0v by Computer Eymen
A simple, fast, and user-friendly command-line calculator application. This tool is developed in C++ to quickly perform basic arithmetic operations.

 Features
Addition (+): Adds two numbers.

Subtraction (-): Subtracts the second number from the first number.

Multiplication (*): Multiplies two numbers.

Division (/): Divides the first number by the second number.

Lightweight Design: Runs on the console, does not consume resources.

 Installation and Usage
You have two options to use this project:

1. Running the Ready-Made Application (Recommended)
Download the compiled .exe file from the Releases section of the project and run it by double-clicking. No installation is required.

2. Compiling from Source Code
If you want to compile the code yourself:

Make sure you have a C++ compiler (GCC, Clang, or Visual Studio).

Open the terminal and use the following command:

Bash
g++ main.cpp -o Calculator
Run the resulting Calculator.exe file.

 Usage Instructions
When you run the program, you will be asked to enter the first number. Type the number and press Enter.

Then enter the second number and press Enter.

Select the operation you want to perform (+, -, *, /) and press Enter.

The result will be displayed on the screen. The application will remain open until you press Enter so that the program does not close immediately.

 Important Notes
If you enter 0 for the second number in the division operation, you may get a mathematically uncertain/infinite result.

The program only supports numerical inputs. Entering letters or symbols may lead to unexpected results.

⚠️Important: Regarding Security Warnings
When you download the application, Windows or your browser may give a "Not Secure" warning. This is because:

The application is not signed with a digital certificate (as I am an individual developer).

The program is completely open source; you can see what it does in the main.cpp code above.

If you receive a warning: You can continue by clicking "Additional Information" -> "Run Anyway".

3. Solutions (Technical)
Use Zip: Don't share the .exe file directly. Put it in a .zip or .rar archive. This sometimes softens the "dangerous file" filter of browsers during download.

Use GitHub Releases: If you upload the file to GitHub's own "Releases" section instead of Discord or file-sharing sites, Google and Microsoft services trust it a little more.

False Positive Reporting: If an antivirus program (such as Avast or Windows Defender) deletes a file, you can report a "False Positive" through their website, but this can be very cumbersome for a small project.
